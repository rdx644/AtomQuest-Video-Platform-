# AtomQuest Video Support Platform

A real-time video calling platform for customer support teams. Built for the AtomQuest Hackathon 1.0.

## Features

### Must-Have
- **Session Management**: Agents can create sessions and generate invite links. Customers can join via the browser.
- **Audio & Video Calling**: Real-time WebRTC media routed through a custom signaling server.
- **In-Call Chat**: Real-time text messaging between participants.
- **User Roles & Access**: Strict separation between Agent (creator, can record, can end) and Customer (joiner only).

### Good-to-Have (Implemented)
- **Call Recording**: Agents can start and stop recordings during a session.
- **File Sharing in Chat**: Participants can upload and share images and documents during the call.
- **Reconnect Handling**: If a customer disconnects, the session remains active for a grace period (120 seconds by default) allowing them to reconnect without losing state.
- **Admin Dashboard**: Live session monitoring and the ability for admins to force-end sessions.
- **Observability**: Prometheus-compatible `/metrics` endpoint for operational monitoring (active sessions, connected participants, API response times).
- **Session History**: Ended sessions and their chats are saved and viewable.

## Setup Steps

### Prerequisites
- Node.js (v18 or higher recommended)
- npm or yarn

### 1. Install Dependencies
Open two terminal windows.
Terminal 1 (Server):
```bash
cd server
npm install
```

Terminal 2 (Client):
```bash
cd client
npm install
```

### 2. Start the Application
Terminal 1 (Server):
```bash
npm run dev
```
The server will start on `http://localhost:3001`. It uses a file-based JSON database (`server/data/database.json`), so no external database setup is required.

Terminal 2 (Client):
```bash
npm run dev
```
The client will start on `http://localhost:5173`.

## How to Test the End-to-End Flow
1. Open `http://localhost:5173` in your browser.
2. Log in as `agent1`.
3. Click **+ New Session** from the dashboard.
4. Once in the call room, click **🔗 Invite** or copy the URL (e.g., `http://localhost:5173/join/<token>`).
5. Open an **Incognito Window** or a different browser and paste the invite link.
6. Enter a name and join the session as the Customer.
7. Both Agent and Customer will now be connected with audio, video, and chat.
8. To test the Admin Dashboard, log in as `admin` in another window to view live sessions.

## Known Limitations
- **Media Routing**: WebRTC signaling is handled by the custom WebSocket server, but the actual media streams are P2P. A true SFU (Selective Forwarding Unit) like Mediasoup was not implemented due to native module build constraints on the deployment environment. However, the signaling server maintains full control over session state and connection establishment.
- **File Storage**: Uploads and recordings are stored locally on the server filesystem (`/uploads` and `/recordings`). In a production environment, this should be migrated to cloud storage (e.g., AWS S3).
- **Database**: Uses a file-based JSON store for simplicity and ease of setup during the hackathon. It is not suitable for high-concurrency production workloads.
